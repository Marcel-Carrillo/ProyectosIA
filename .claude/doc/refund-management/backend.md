## Architecture Overview

This feature adds admin refund CRUD with a state machine and `paymentStatus` synchronization, closing the financial lifecycle of a `CustomerOrder`. It follows the exact DDD layered pattern used by `customer-order-management`:

```
Presentation (controller + routes)
    ↓
Application (service + validator)
    ↓
Domain (model + repository interface)
    ↓
Infrastructure (Prisma repository + error classes)
    ↓
Prisma schema (Refund model)
```

## Critical Notes (Read Before Implementing)

1. **No Zod in codebase.** The tasks.md calls for a `presentation/validators/refundValidators.ts` with Zod schemas, but Zod is NOT installed (`package.json` has no Zod dependency). All validation MUST follow the existing pattern: manual functions added to `backend/src/application/validator.ts`. Do NOT install Zod; do NOT create a `presentation/validators/` directory.

2. **No `requireAdminAuth` middleware exists.** The session context and codebase confirm this. Routes must NOT reference this middleware. Admin routes are currently unguarded (same as all other admin routes in this codebase).

3. **Error classes go in the infrastructure repository file.** Pattern: all error classes (`RefundNotFoundError`, `RefundOrderNotPaidError`, etc.) are defined and exported from `backend/src/infrastructure/repositories/refundRepository.ts`, then imported into the errorHandler and service.

4. **`Decimal.js` import.** Use `import { Decimal } from '@prisma/client/runtime/library'`. This is the pattern in `customerOrderService.ts`.

5. **Interactive Prisma transactions.** The `createRefund` and `updateRefundStatus` service methods require `prisma.$transaction(async (tx) => { ... })` (interactive transaction) — NOT the array-style `prisma.$transaction([...])` used by `findAll`. This is because the validation logic depends on data read within the same transaction.

6. **Decimal storage as string.** All monetary amounts on the domain model are stored as `string` (using `.toString()` on the Prisma `Decimal`), exactly as `CustomerOrder.totalAmount`, `subtotalAmount`, etc.

7. **`returnRequestId` has NO Prisma `@relation`.** Store as `Int?` with no `@relation` annotation. KAN-25 will add the FK constraint. No existence check for `returnRequestId` until KAN-25 lands.

8. **`paymentStatus` sync logic.** Only `Completed` refunds drive `paymentStatus` updates. The amount validation cap uses `Completed + Processing` (to prevent over-commitment), but the `recalculatePaymentStatus` helper only sums `Completed` refunds.

9. **`processedAt`** is set to `new Date()` only when transitioning to `Completed`, inside the transaction.

```typescript
export type RefundStatus =
  | 'Pending'
  | 'Processing'
  | 'Completed'
  | 'Failed'
  | 'Cancelled';

export const REFUND_TRANSITIONS: Record<RefundStatus, RefundStatus[]> = {
  Pending: ['Processing', 'Cancelled'],
  Processing: ['Completed', 'Failed', 'Cancelled'],
  Completed: [],
  Failed: [],
  Cancelled: [],
};

export function isValidRefundTransition(from: RefundStatus, to: RefundStatus): boolean {
  return REFUND_TRANSITIONS[from]?.includes(to) ?? false;
}

export class Refund {
  id?: number;
  customerOrderId: number;
  returnRequestId: number | null;
  amount: string;
  reason: string | null;
  status: RefundStatus;
  paymentProviderReference: string | null;
  createdAt?: Date;
  updatedAt?: Date;
  processedAt: Date | null;

  constructor(data: {
    id?: number;
    customerOrderId: number;
    returnRequestId?: number | null;
    amount: string | number | { toString(): string };
    reason?: string | null;
    status?: string;
    paymentProviderReference?: string | null;
    createdAt?: Date;
    updatedAt?: Date;
    processedAt?: Date | null;
  }) {
    this.id = data.id;
    this.customerOrderId = data.customerOrderId;
    this.returnRequestId = data.returnRequestId ?? null;
    this.amount = String(data.amount);
    this.reason = data.reason ?? null;
    this.status = (data.status as RefundStatus) ?? 'Pending';
    this.paymentProviderReference = data.paymentProviderReference ?? null;
    this.createdAt = data.createdAt;
    this.updatedAt = data.updatedAt;
    this.processedAt = data.processedAt ?? null;
  }
}
```

**Key design notes:**
- `amount` is stored as `string` to match the monetary-value convention used by `CustomerOrder`.
- `status` is `string` in Prisma but typed as `RefundStatus` in the domain.
- `REFUND_TRANSITIONS` and `isValidRefundTransition` are pure domain logic with no framework dependencies.
- Terminal states (`Completed`, `Failed`, `Cancelled`) have empty arrays in `REFUND_TRANSITIONS`.

**`RefundRepository` class:**

```typescript
export class RefundRepository implements IRefundRepository {
  async findAll(filters: RefundListFilters): Promise<RefundListResult> {
    const page = filters.page && filters.page >= 1 ? filters.page : 1;
    const limit = filters.limit && filters.limit >= 1 ? Math.min(filters.limit, 100) : 10;
    const skip = (page - 1) * limit;

    const where: Prisma.RefundWhereInput = {};
    if (filters.customerOrderId) where.customerOrderId = filters.customerOrderId;
    if (filters.status) where.status = filters.status;

    const [rows, total] = await prisma.$transaction([
      prisma.refund.findMany({
        where,
        orderBy: { createdAt: 'desc' },
        skip,
        take: limit,
        select: refundSelect,
      }),
      prisma.refund.count({ where }),
    ]);

    return { items: rows.map(mapRefund), total, page, limit };
  }

  async findById(id: number): Promise<Refund | null> {
    const row = await prisma.refund.findUnique({
      where: { id },
      select: refundSelect,
    });
    return row ? mapRefund(row) : null;
  }

  async create(data: RefundCreateData): Promise<Refund> {
    const row = await prisma.refund.create({
      data: {
        customerOrderId: data.customerOrderId,
        returnRequestId: data.returnRequestId ?? null,
        amount: data.amount,
        reason: data.reason ?? null,
        paymentProviderReference: data.paymentProviderReference ?? null,
        status: 'Pending',
      },
      select: refundSelect,
    });
    return mapRefund(row);
  }

  async updateStatus(id: number, data: RefundStatusUpdateData): Promise<Refund> {
    try {
      const row = await prisma.refund.update({
        where: { id },
        data: {
          status: data.status,
          ...(data.processedAt !== undefined && { processedAt: data.processedAt }),
          ...(data.paymentProviderReference !== undefined && {
            paymentProviderReference: data.paymentProviderReference,
          }),
        },
        select: refundSelect,
      });
      return mapRefund(row);
    } catch (err) {
      if (err instanceof Prisma.PrismaClientKnownRequestError && err.code === 'P2025') {
        throw new RefundNotFoundError();
      }
      throw err;
    }
  }
}
```

**Important:** The `create` and `updateStatus` methods on the repository itself use `prisma.refund.create/update` directly (not inside a transaction). The service wraps calls inside `prisma.$transaction(async (tx) => {...})` and passes the transaction client `tx` — but because the repository is designed to work with the global `prisma` client, the service bypasses the repository for transactional operations and calls `tx.refund.create` / `tx.refund.update` / `tx.customerOrder.update` directly inside the transaction. See Service section below for how this is handled.

**Class definition:**

```typescript
const MAX_LIMIT = 100;

export class RefundService {
  constructor(private readonly repo: IRefundRepository) {}

  async listRefunds(filters: RefundListFilters = {}): Promise<RefundListResult> {
    const limit =
      filters.limit !== undefined
        ? Math.min(Math.max(1, filters.limit), MAX_LIMIT)
        : 10;
    return this.repo.findAll({ ...filters, limit });
  }

  async getRefundById(id: number): Promise<Refund> {
    const refund = await this.repo.findById(id);
    if (!refund) throw new RefundNotFoundError();
    return refund;
  }

  async createRefund(data: RefundCreateData): Promise<Refund> {
    // 1. Input validation (throws ValidationError on bad input)
    validateRefundCreateData(data as unknown as Record<string, unknown>);

    return prisma.$transaction(async (tx) => {
      // 2. Fetch order inside transaction
      const order = await tx.customerOrder.findUnique({
        where: { id: data.customerOrderId },
        select: { id: true, totalAmount: true, paymentStatus: true },
      });
      if (!order) throw new CustomerOrderNotFoundError();

      // 3. Validate paymentStatus
      if (!['Paid', 'PartiallyRefunded'].includes(order.paymentStatus)) {
        throw new RefundOrderNotPaidError();
      }

      // 4. Sum Completed + Processing refunds inside transaction
      const aggregate = await tx.refund.aggregate({
        where: {
          customerOrderId: data.customerOrderId,
          status: { in: ['Completed', 'Processing'] },
        },
        _sum: { amount: true },
      });
      const committed = new Decimal(aggregate._sum.amount?.toString() ?? '0');
      const totalAmount = new Decimal(order.totalAmount.toString());
      const refundable = totalAmount.sub(committed);
      const requestedAmount = new Decimal(String(data.amount));

      if (requestedAmount.greaterThan(refundable)) {
        throw new RefundAmountExceedsBalanceError();
      }

      // 5. Create refund
      const refundRow = await tx.refund.create({
        data: {
          customerOrderId: data.customerOrderId,
          returnRequestId: data.returnRequestId ?? null,
          amount: requestedAmount.toFixed(2),
          reason: data.reason ?? null,
          paymentProviderReference: data.paymentProviderReference ?? null,
          status: 'Pending',
        },
      });

      // 6. Recalculate paymentStatus (no-op for Pending, but called for consistency)
      await this.recalculatePaymentStatus(data.customerOrderId, tx);

      return new Refund({ ...refundRow, amount: refundRow.amount.toString() });
    });
  }

  async updateRefundStatus(
    id: number,
    newStatus: RefundStatus,
    paymentProviderReference?: string | null
  ): Promise<Refund> {
    // 1. Input validation
    validateRefundStatusUpdate({ status: newStatus, paymentProviderReference } as unknown as Record<string, unknown>);

    return prisma.$transaction(async (tx) => {
      // 2. Load refund inside transaction
      const current = await tx.refund.findUnique({ where: { id } });
      if (!current) throw new RefundNotFoundError();

      // 3. Validate transition
      if (!isValidRefundTransition(current.status as RefundStatus, newStatus)) {
        throw new RefundTransitionInvalidError(
          `Cannot transition refund from ${current.status} to ${newStatus}`
        );
      }

      // 4. Prepare update data
      const processedAt = newStatus === 'Completed' ? new Date() : undefined;

      const updatedRow = await tx.refund.update({
        where: { id },
        data: {
          status: newStatus,
          ...(processedAt !== undefined && { processedAt }),
          ...(paymentProviderReference !== undefined && { paymentProviderReference }),
        },
      });

      // 5. Recalculate paymentStatus on any terminal transition
      if (['Completed', 'Cancelled', 'Failed'].includes(newStatus)) {
        await this.recalculatePaymentStatus(current.customerOrderId, tx);
      }

      return new Refund({ ...updatedRow, amount: updatedRow.amount.toString() });
    });
  }

  private async recalculatePaymentStatus(
    customerOrderId: number,
    tx: Parameters<Parameters<typeof prisma.$transaction>[0]>[0]
  ): Promise<void> {
    const order = await tx.customerOrder.findUnique({
      where: { id: customerOrderId },
      select: { totalAmount: true },
    });
    if (!order) return;

    const aggregate = await tx.refund.aggregate({
      where: { customerOrderId, status: 'Completed' },
      _sum: { amount: true },
    });

    const completedSum = new Decimal(aggregate._sum.amount?.toString() ?? '0');
    const totalAmount = new Decimal(order.totalAmount.toString());

    let newPaymentStatus: string | undefined;
    if (completedSum.equals(totalAmount) && completedSum.greaterThan(0)) {
      newPaymentStatus = 'Refunded';
    } else if (completedSum.greaterThan(0)) {
      newPaymentStatus = 'PartiallyRefunded';
    }
    // If completedSum == 0, leave paymentStatus unchanged (Cancelled/Failed scenario)

    if (newPaymentStatus) {
      await tx.customerOrder.update({
        where: { id: customerOrderId },
        data: { paymentStatus: newPaymentStatus },
      });
    }
  }
}
```

```typescript
const REFUND_STATUSES = ['Pending', 'Processing', 'Completed', 'Failed', 'Cancelled'] as const;

export function validateRefundCreateData(data: Record<string, unknown>): void {
  const customerOrderId = data['customerOrderId'];
  if (customerOrderId === undefined || customerOrderId === null) {
    throw new ValidationError("Field 'customerOrderId' is required");
  }
  if (!Number.isInteger(customerOrderId) || (customerOrderId as number) < 1) {
    throw new ValidationError("Field 'customerOrderId' must be a positive integer");
  }

  const amount = data['amount'];
  if (amount === undefined || amount === null || amount === '') {
    throw new ValidationError("Field 'amount' is required");
  }
  const amountNum = Number(amount);
  if (!Number.isFinite(amountNum) || amountNum <= 0) {
    throw new ValidationError("Field 'amount' must be a positive number");
  }

  const returnRequestId = data['returnRequestId'];
  if (returnRequestId !== undefined && returnRequestId !== null) {
    if (!Number.isInteger(returnRequestId) || (returnRequestId as number) < 1) {
      throw new ValidationError("Field 'returnRequestId' must be a positive integer");
    }
  }

  const reason = data['reason'];
  if (reason !== undefined && reason !== null && reason !== '') {
    if (typeof reason === 'string' && reason.length > 500) {
      throw new ValidationError("Field 'reason' must not exceed 500 characters");
    }
  }

  const paymentProviderReference = data['paymentProviderReference'];
  if (paymentProviderReference !== undefined && paymentProviderReference !== null && paymentProviderReference !== '') {
    if (typeof paymentProviderReference === 'string' && paymentProviderReference.length > 150) {
      throw new ValidationError("Field 'paymentProviderReference' must not exceed 150 characters");
    }
  }
}

export function validateRefundStatusUpdate(data: Record<string, unknown>): void {
  const status = data['status'];
  if (status === undefined || status === null || status === '') {
    throw new ValidationError("Field 'status' is required");
  }
  if (!REFUND_STATUSES.includes(status as (typeof REFUND_STATUSES)[number])) {
    throw new ValidationError(
      `Field 'status' must be one of: ${REFUND_STATUSES.join(', ')}`
    );
  }

  const paymentProviderReference = data['paymentProviderReference'];
  if (paymentProviderReference !== undefined && paymentProviderReference !== null && paymentProviderReference !== '') {
    if (typeof paymentProviderReference === 'string' && paymentProviderReference.length > 150) {
      throw new ValidationError("Field 'paymentProviderReference' must not exceed 150 characters");
    }
  }
}
```

```typescript
import { Request, Response, NextFunction } from 'express';
import { RefundService } from '../../application/services/refundService';

function parseIdParam(value: string): number {
  const id = parseInt(value, 10);
  if (isNaN(id)) {
    throw new ValidationError("Parameter 'id' must be a valid integer");
  }
  return id;
}

const refundService = new RefundService(new RefundRepository());

export async function listRefunds(req: Request, res: Response, next: NextFunction): Promise<void> {
  try {
    const { page, limit, customerOrderId, status } = req.query;
    const result = await refundService.listRefunds({
      page: page ? parseInt(String(page), 10) : undefined,
      limit: limit ? parseInt(String(limit), 10) : undefined,
      customerOrderId: customerOrderId ? parseInt(String(customerOrderId), 10) : undefined,
      status: status as RefundStatus | undefined,

    res.json({ success: true, data: result, message: 'Refunds retrieved successfully' });

  }
```

## Response Envelope Summary

All endpoints return:
- `GET /` → `200 { success: true, data: { items, total, page, limit }, message: '...' }`
- `POST /` → `201 { success: true, data: Refund, message: '...' }`
- `GET /:id` → `200 { success: true, data: Refund, message: '...' }`
- `PATCH /:id/status` → `200 { success: true, data: Refund, message: '...' }`
- Errors → `{ success: false, error: { message, code } }`

HTTP status code map:
| Error class | HTTP | Code |
|---|---|---|
| `ValidationError` | 400 | `VALIDATION_ERROR` |
| `CustomerOrderNotFoundError` | 404 | `CUSTOMER_ORDER_NOT_FOUND` |
| `RefundNotFoundError` | 404 | `REFUND_NOT_FOUND` |
| `RefundOrderNotPaidError` | 409 | `REFUND_ORDER_NOT_PAID` |
| `RefundAmountExceedsBalanceError` | 409 | `REFUND_AMOUNT_EXCEEDS_BALANCE` |
| `RefundTransitionInvalidError` | 409 | `REFUND_TRANSITION_INVALID` |
